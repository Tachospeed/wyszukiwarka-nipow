Infolab Serwis

Next.js 15 Supabase Pipedrive CRM TypeScript Tailwind CSS

Wewnętrzna wyszukiwarka firm dla handlowców. Umożliwia szybkie pobieranie danych o firmie z systemu Pipedrive CRM na podstawie numeru NIP.

📋 O projekcie

Aplikacja webowa zbudowana w Next.js 15 (App Router), służy jako panel dostępu do danych CRM. Użytkownik loguje się emailem i hasłem, wpisuje NIP firmy, a system pobiera z Pipedrive wszystkie powiązane dane: dane organizacji, osoby kontaktowe, deale i opiekunów.

🔍 Główna funkcja

Wyszukiwanie firm po NIP w systemie Pipedrive CRM. Wyniki pokazują organizację, kontakty, deale i opiekunów.

👑 Panel administratora

Zarządzanie kontami użytkowników (role admin/user) i numerami telefonów opiekunów z CRM.

🗺 Routing

ŚcieżkaDostępOpis
GET/ Publiczny Strona logowania. Zalogowani są redirectowani do /dashboard lub /admin.
GET/dashboard Zalogowany Główny widok — formularz wyszukiwania NIP + wyniki z Pipedrive.
GET/dashboard?nip=XXX Zalogowany Automatyczne wyszukiwanie po przejściu z parametrem NIP w URL.
GET/admin Admin tylko Panel zarządzania użytkownikami (role).
GET/admin/phones Admin tylko Zarządzanie numerami telefonów opiekunów z CRM.
ℹ️

Middleware w middleware.ts automatycznie chroni trasy — użytkownik bez sesji jest redirectowany na /, a zwykły użytkownik próbujący wejść na /admin trafia do /dashboard?error=unauthorized_admin_access.

🔑 Zmienne środowiskowe

Utwórz plik .env.local w katalogu głównym projektu. Nie commituj go do gita — plik jest już w .gitignore.

# ── Supabase ────────────────────────────────────────────────────
NEXT_PUBLIC_SUPABASE_URL=https://twoj-projekt.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIs...
SUPABASE_SERVICE_ROLE_KEY=eyJhbGciOiJIUzI1NiIs...

# ── Pipedrive ───────────────────────────────────────────────────
PIPEDRIVE_API_KEY=twoj_klucz_api_pipedrive

# Opcjonalne — domyślnie https://api.pipedrive.com/api
PIPEDRIVE_API_BASE_URL=https://api.pipedrive.com/api

# Opcjonalne — włącza szczegółowe logi w konsoli serwera
PIPEDRIVE_DEBUG=true

Gdzie znaleźć wartości?

ZmiennaGdzie szukać
NEXT_PUBLIC_SUPABASE_URL Supabase Dashboard → Project Settings → API → Project URL
NEXT_PUBLIC_SUPABASE_ANON_KEY Supabase Dashboard → Project Settings → API → anon public key
SUPABASE_SERVICE_ROLE_KEY Supabase Dashboard → Project Settings → API → service_role key (⚠️ sekret!)
PIPEDRIVE_API_KEY Pipedrive → kliknij avatar → Personal preferences → API → Your personal API token
🔐

SUPABASE_SERVICE_ROLE_KEY omija wszystkie reguły RLS i daje pełny dostęp do bazy. Nigdy nie umieszczaj go w kodzie frontendowym ani nie ujawniaj publicznie.

🗄 Baza danych (Supabase)

Projekt wymaga dwóch tabel w Supabase. Skrypty SQL są w folderze scripts/ — należy je uruchomić w kolejności numerycznej w Supabase SQL Editor.

Otwórz Supabase SQL Editor W panelu Supabase: lewe menu → SQL Editor → kliknij + New query.
Uruchom skrypt 001 — tabela ról Wklej i uruchom zawartość pliku scripts/001-create-user-roles.sql. Tworzy tabelę user_roles, polityki RLS i funkcję is_current_user_admin().
Uruchom skrypt 002 — auto-admin Wklej i uruchom scripts/002-auto-admin-first-user.sql. Nadaje rolę admina pierwszemu zarejestrowanemu użytkownikowi (jeśli żaden admin jeszcze nie istnieje).
Uruchom skrypt 003 — poprawki polityk RLS Wklej i uruchom scripts/003-fix-user-roles-policies.sql. Naprawia polityki bezpieczeństwa dla tabeli ról.
Uruchom skrypt 006 — tabela telefonów Wklej i uruchom scripts/006-create-owner-phones-table.sql. Tworzy tabelę owner_phone_numbers do przechowywania numerów telefonów opiekunów z CRM.
⚠️

Skryptów 005-add-phone-to-users.sql i 005-revert-phone-from-users.sql nie uruchamiaj — to stare, zastąpione migracje (numer 005 był używany dwa razy). Skrypt 006 jest aktualną wersją obsługi telefonów.

Schemat bazy

TabelaKolumnyOpis
user_roles user_id, role, created_at, updated_at Role użytkowników (admin lub user). RLS włączone.
owner_phone_numbers email, phone_number, created_at, updated_at Numery telefonów opiekunów z CRM, identyfikowane emailem. RLS wyłączone (zarządzane przez admin backend).

🚀 Deployment na Vercel

Wgraj kod na GitHub / GitLab / Bitbucket 
$ git init && git add . && git commit -m "Initial commit"
$ git remote add origin https://github.com/twoja-org/infolab-serwis.git
$ git push -u origin main
Połącz repozytorium z Vercel Zaloguj się na vercel.com → Add New Project → wybierz repozytorium. Vercel automatycznie wykryje Next.js.
Ustaw zmienne środowiskowe w Vercel W kroku konfiguracji (lub później: Settings → Environment Variables) dodaj wszystkie zmienne z pliku .env.local:
  • NEXT_PUBLIC_SUPABASE_URL
  • NEXT_PUBLIC_SUPABASE_ANON_KEY
  • SUPABASE_SERVICE_ROLE_KEY
  • PIPEDRIVE_API_KEY
  • PIPEDRIVE_API_BASE_URL (opcjonalnie)
Kliknij Deploy Vercel zbuduje aplikację poleceniem pnpm build i opublikuje ją. Każdy push na main wyzwoli automatyczny redeploy.
Skonfiguruj domenę (opcjonalnie) Vercel → Settings → Domains → dodaj własną domenę. SSL jest generowany automatycznie.
💡

Jeśli chcesz budować lokalnie przed push-em, użyj pnpm build && pnpm start. Vercel uruchamia dokładnie te same polecenia.

👑 Nadawanie roli administratora

Automatycznie (zalecane przy pierwszym uruchomieniu)

Po uruchomieniu skryptu 002-auto-admin-first-user.sql — pierwszy zarejestrowany użytkownik automatycznie otrzymuje rolę admina.

Ręcznie przez SQL Editor

Wejdź do Supabase → Authentication → Users, znajdź UUID użytkownika i uruchom:

INSERT INTO public.user_roles (user_id, role)
VALUES ('uuid-uzytkownika-tutaj', 'admin')
ON CONFLICT (user_id) DO UPDATE SET role = 'admin';

Przez panel admina (jeśli masz już admina)

Zaloguj się jako admin → kliknij pomarańczowy przycisk Panel Admina → przy wybranym użytkowniku zmień rolę na Admin.

⚠️

Jeśli utracisz dostęp do konta administratora, jedynym sposobem odzyskania go jest metoda przez SQL Editor (wymaga dostępu do panelu Supabase).

👥 Zarządzanie użytkownikami

Panel dostępny pod /admin — tylko dla adminów.

Co możesz zrobić w panelu admin?
  • Przeglądać listę wszystkich zarejestrowanych użytkowników
  • Zmienić rolę użytkownika: Admin ↔ User
  • Usunąć użytkownika z systemu
ℹ️

Rejestracja nowych kont odbywa się przez stronę logowania — użytkownicy sami się rejestrują (lub admin może to zrobić za nich). Nowy użytkownik domyślnie otrzymuje rolę user.

📞 Numery telefonów opiekunów

Panel dostępny pod /admin/phones.

System Pipedrive często nie przechowuje numerów telefonów użytkowników CRM (opiekunów). Ta funkcja pozwala adminowi ręcznie przypisać numer telefonu do emaila opiekuna — będzie on wyświetlany zamiast "Brak danych" przy wynikach wyszukiwania.

OperacjaOpis
Dodaj telefon Podaj email opiekuna z Pipedrive i jego numer telefonu
Edytuj telefon Zmień istniejący numer dla danego emaila
Usuń telefon Usuwa wpis — przy wyszukiwaniu znów pojawi się "Brak danych"

🔍 Jak działa wyszukiwarka NIP

Wpisz NIP firmy (bez kresek) i kliknij Wyszukaj. System wykona następujące kroki:

Wyszukanie organizacji Zapytanie do Pipedrive API /v1/organizations/search z parametrem term=NIP i exact_match=1.
Pobranie osób kontaktowych Dla każdej organizacji pobiera listę osób (/v1/organizations/{id}/persons).
Pobranie dealów Zapytanie do /v2/deals?org_id={id} — zwraca wszystkie deale powiązane z organizacją.
Pobranie opiekunów Dla każdego dealu i organizacji pobiera dane opiekuna (/v1/users/{id}), a numer telefonu nadpisuje danymi z tabeli owner_phone_numbers.
⏱

Wyszukiwanie może trwać kilka sekund — API Pipedrive wykonuje wiele kolejnych zapytań (osoby, deale, pipeline, opiekunowie). To normalne zachowanie.

🐛 Tryb debug (Pipedrive)

Ustaw zmienną PIPEDRIVE_DEBUG=true w .env.local, aby włączyć szczegółowe logowanie w konsoli serwera Next.js. Logi pokazują:

  • Każde zapytanie HTTP do Pipedrive (URL bez klucza API, czas odpowiedzi)
  • Status i nagłówki odpowiedzi
  • Przetworzone dane (organizacje, osoby, deale)
  • Ostrzeżenia (brak pipeline, brak stage, itp.)
🚫

Nigdy nie włączaj PIPEDRIVE_DEBUG=true na produkcji. Logi mogą zawierać wrażliwe dane biznesowe i spowalniają odpowiedzi serwera.

❓ FAQ

Nie mogę się zalogować — "Invalid login credentials"

Sprawdź czy użytkownik istnieje w Supabase Dashboard → Authentication → Users. Jeśli nie — zarejestruj konto przez stronę logowania.

Po zalogowaniu widzę dashboard, ale wyszukiwanie nie działa

Sprawdź zmienną PIPEDRIVE_API_KEY — czy jest ustawiona i czy nie zawiera placeholdera. Włącz PIPEDRIVE_DEBUG=true i sprawdź logi w terminalu.

Strona /admin przekierowuje mnie na dashboard

Twoje konto nie ma roli admin. Sprawdź tabelę user_roles w Supabase lub nadaj rolę ręcznie przez SQL Editor (patrz sekcja Pierwszy admin).

Build na Vercel kończy się błędem TypeScript

Uruchom lokalnie pnpm build — zobaczysz dokładny błąd. Najczęstsza przyczyna to brakujące zmienne środowiskowe podczas buildu.

Pipedrive zwraca "Brak danych" przy telefonie opiekuna

To normalne — Pipedrive często nie przechowuje telefonów użytkowników CRM. Dodaj numer ręcznie w panelu /admin/phones.

Jak dodać nowego użytkownika bez dostępu do panelu admin?

Użytkownik może się sam zarejestrować na stronie logowania (/ → kliknij link rejestracji). Domyślnie dostaje rolę user.
